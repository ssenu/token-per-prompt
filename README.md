# token-per-prompt 📊

> **질문 하나가 토큰을 얼마나 썼는지** 보여주는 Claude Code 플러그인
> — 세션 총량 대시보드는 많지만, "이 질문 하나"의 비용을 보여주는 건 이게 처음입니다.

100% 로컬에서 동작하고(대화 기록만 읽음), **토큰을 전혀 소모하지 않으며**, macOS·Windows 모두에서 외부 라이브러리 없이 파이썬 표준 라이브러리만으로 돌아갑니다.

## 1. 무엇을 하는 도구인가

Claude Code는 화면 오른쪽 아래에 **현재 컨텍스트 창 크기**를, 대시보드는 **세션 누적/요금제 %**를 보여줍니다. 하지만 **"방금 그 질문 하나가 얼마나 썼는지"**는 어디에도 안 나옵니다.

이 플러그인은 바로 그 **질문(턴) 단위 사용량**을 세 곳에 보여줍니다:

| 방식 | 위치 | 성격 |
|------|------|------|
| 🧮 **훅** | 답변 바로 아래 | 그 질문의 값을 **고정 기록** |
| 📊 **상태줄** | 화면 하단 | 현재 턴 **실시간**(🔄 진행중 / ✅ 완료) |
| 📋 **리포트** | 명령 실행 시 | 세션 **전체 질문 표** |

---

## 2. 설치

### 사전 요구
- Claude Code
- 파이썬 3.8+ (`python --version` 또는 `python3 --version`)

### 플러그인 설치
```bash
# 1) 마켓플레이스 등록
claude plugin marketplace add <GitHub사용자명>/token-per-prompt

# 2) 전역 설치
claude plugin install token-per-prompt@token-per-prompt -s user
```

> 설치만으로는 아직 화면에 안 뜹니다. 아래 `/token-setup`으로 **어디에 어떻게 표시할지**를 한 번 정해줘야 합니다.

## 3. 빠른 시작

Claude Code 안에서:

```
/token-setup
```
→ 표시 위치·수준·요금제·테마 등을 물어봅니다. 답하면 자동으로 설정되고, **Claude Code를 재시작**하면 적용됩니다.

이후:
```
/token-report          # 이번 세션 질문별 토큰 표
```
또는 자연어로 **"토큰 사용량 표 보여줘"**, **"토큰 스킬 설정해줘"** 라고 해도 됩니다.

## 4. 세 가지 표시 방식

### 🧮 답변 아래 (훅)
매 답변이 끝나면 그 질문의 토큰이 한 줄로 남습니다. 스크롤해도 그대로 — **그 질문의 확정 기록**입니다.
```
출력 1,234 · 새 컨텍스트 80 · 캐시읽기 320,000 | 5시간 리밋의 약 0.25%
```

### 📊 상태줄 (실시간)
화면 하단에 현재 턴 값이 실시간으로 뜹니다. 질문마다 **0으로 리셋 → 단계별로 오름**.
```
🔄 출력 890 · 새 컨텍스트 60 · 캐시읽기 300,000 | 5시간 리밋의 약 0.18%   ← 진행중
✅ 출력 1,234 · 새 컨텍스트 80 · 캐시읽기 320,000 | 5시간 리밋의 약 0.25%   ← 완료
```
- 맨 앞 **🔄 = 아직 오르는 중**, **✅ = 값 확정**.
- 이미 다른 상태줄(예: claude-dashboard)을 쓰고 있으면, 그 줄은 **우리 줄 위에 그대로 유지**됩니다.

### 📋 리포트 표
`/token-report` → [7번 섹션](#7-리포트-표) 참고.

## 5. 설정 항목

`/token-setup`에서 물어보는 것들:

| 항목 | 값 | 설명 |
|------|-----|------|
| **표시 위치** | `hook` / `statusline` / `both` | 답변 아래만 / 상태줄만 / 둘 다 |
| **표시 수준** | `simple` / `basic` / `detailed` | 총토큰만 / 종류별 / 종류별+5시간% |
| **요금제** | `pro` / `max5` / `max20` | 5시간 % 계산 기준 |
| **테마** | `tokyonight` / `default` | 상태줄 색상 (default=무색) |
| **5시간 %** | `on` / `off` | % 표시 여부 |
| **자동보정** | `on` / `off` | 서버 실측으로 리밋 보정 ([8번](#8-자동보정은-어떻게-동작하나)) |
| **새로고침** | 1 / 2 / 5초 | 상태줄 갱신 주기 |

- 설정 파일: `~/.claude/token-per-prompt/config.json`
- 배선(훅·상태줄): `~/.claude/settings.json`

> **답변 아래만** 쓰고 싶으면 위치를 `hook`으로 두세요. 상태줄 관련(색·새로고침·대시보드 체이닝)은 전부 건너뜁니다.

## 6. 숫자 읽는 법 (중요)

### 토큰 종류

| 표시 | API | 실제 부담 |
|------|-----|-----------|
| **출력** | output | 생성한 답변, **제일 비쌈** |
| **새 컨텍스트** | cache write | 처음 처리해 캐시에 씀 (입력의 ~1.25배) |
| **캐시읽기** | cache read | 기존 맥락 재읽기, **정가의 ~10%** (싸지만 0은 아님) |

👉 **실질 소비 = 출력 + 새 컨텍스트.** 캐시읽기는 대화가 길수록 숫자가 커지지만 비용 부담은 미미합니다.

### 5시간 %
```
이 질문 % = (출력 + 새 컨텍스트 + 입력) ÷ 5시간 리밋 × 100     (캐시읽기 제외)
```
리밋은 요금제별 **추정 기본값**이며, 자동보정을 켜면 실제 서버 값으로 맞춰집니다. **정확한 공식 수치가 아니라 근사치**입니다.

### 상태줄 값 ≠ 답변 아래 값? — 정상입니다
- **답변 아래** = 그 질문이 끝난 순간의 **고정 기록**
- **상태줄** = 항상 **최신 턴의 실시간 값**

둘은 **같은 턴을 가리킬 때만**(상태줄이 ✅이고 그 질문일 때) 일치합니다. 상태줄이 🔄(진행중)이거나 이미 다음 질문으로 넘어갔으면 값이 달라 보이는 게 당연합니다.

> 비유: **답변 아래 = 영수증(고정), 상태줄 = 금전등록기 화면(실시간)**. 총액은 영수증(질문별 값)을 더하면 됩니다.

## 7. 리포트 표

```
/token-report
```
현재 세션의 모든 질문을 한 표로:

```
시각           출력       새컨텍스트     5시간%   질문
────────────────────────────────────────────────────────────────────
14:06     5,165      12,335    2.92%   질문별로 토큰 사용량 보여줘
13:48     2,823     103,409   17.71%   노트북lm 사용법 알려줘
────────────────────────────────────────────────────────────────────
합계        7,988     115,744   20.63%   (2개 질문)

※ % = (출력+새컨텍스트+입력) ÷ 5시간리밋 · 캐시읽기 제외
```
- 특정 세션: `summary.py --session <transcript.jsonl 경로>`
- 여러 세션을 동시에 열어두면 자동탐지가 헷갈릴 수 있으니, 그럴 땐 `--session`으로 지정하세요.

## 8. 자동보정은 어떻게 동작하나

Anthropic의 실제 5시간 사용률(`oauth/usage`)은 **1% 단위**라 질문 하나(보통 1% 미만)를 직접 표시하기엔 너무 거칩니다. 그래서 **직접 표시 대신 리밋 보정에 사용**합니다:

```
사용률이 Δ% 오르는 동안 쓴 토큰이 T라면 → 실제 5시간 리밋 ≈ T ÷ (Δ/100)
```

- 이렇게 역산한 리밋을 지수평활로 계속 갱신 → 질문별 %가 **토큰 기반의 세밀함**을 유지하면서 **실제에 맞춰집니다**.
- 서버 호출은 **5분 캐시** → 매 턴 돌아도 실제 요청은 5분에 한 번 (rate limit 안전).
- **토큰 소모 0** (계정 통계 조회는 모델 호출이 아님).
- 한계: 여러 세션 동시 사용 시 사용률이 섞여 보정이 흔들릴 수 있음(단일 세션이면 정확).

## 9. 자주 묻는 질문

**Q. 설치하면 토큰을 더 쓰나요?**
아니요. 전부 로컬 파일을 읽는 것이라 **모델 호출이 없어 토큰 0**입니다. 자동보정의 서버 조회도 계정 통계라 토큰과 무관합니다.

**Q. 상태줄 숫자가 실시간으로 "부드럽게" 오르나요?**
단계(step) 단위로 오릅니다. 토큰은 각 메시지(툴 호출·답변)가 끝날 때 기록되므로, 툴을 여러 번 쓰는 답변은 계단식으로 오르고, 짧은 답변은 끝에 한 번 점프합니다.

**Q. 대시보드의 In/Out/W/R 과 뭐가 다른가요?**
종류(입력/출력/캐시)는 같지만 **범위가 다릅니다**. 대시보드는 "현재 컨텍스트 창(최근 호출 1개)", 우리는 "이 질문 전체(모든 단계 합)". 그래서 툴을 많이 쓴 답변일수록 우리 숫자가 더 큽니다.

**Q. 답변 아래만 쓰고 상태줄은 끄고 싶어요.**
`/token-setup`에서 위치를 `hook`으로 선택하세요.

## 10. 문제 해결

| 증상 | 확인 |
|------|------|
| 답변 아래에 안 뜸 | `~/.claude/settings.json`의 `hooks.Stop`에 report.py가 있는지 / Claude Code 재시작 |
| 상태줄에 안 뜸 | `statusLine.command`가 statusline.py인지 / 재시작 / 잠깐(새로고침 주기) 대기 |
| `python` 인식 안 됨 | 파이썬 설치 및 PATH 확인. macOS/Linux는 `python3` |
| 완료됐는데 계속 🔄 | 플러그인 최신 버전인지 확인 (이 이슈는 v0.1.0에서 해결됨) |
| % 가 실제와 차이 큼 | 자동보정을 `on`으로 (기본값). 초반 몇 분은 부트스트랩이라 기본 추정값 사용 |
| 경로가 바뀌어 안 됨 | 플러그인 업데이트 후 `/token-setup` 재실행 (절대경로 재기록) |

---

## 11. 제거

```bash
claude plugin uninstall token-per-prompt@token-per-prompt
```
그 뒤 `~/.claude/settings.json`에서 token-per-prompt 관련 `hooks.Stop` 항목과 `statusLine`을 정리하고(원하면 이전 상태줄로 복원), `~/.claude/token-per-prompt/` 폴더를 삭제하면 됩니다.
